## 概述

IT 服务管理（ITSM）是一种基于流程的实践，旨在使 IT 服务的交付与企业的需求保持一致。其中一部分是管理服务目录，该目录定义了与终端用户（或客户）之间的服务、SLA 和合同。

此模块允许服务经理定义服务目录的所有组成部分：

- 服务
- 服务子类别
- SLA 和 SLT
- 客户合同
- 供应商合同

服务管理与工单管理系统集成：

- 为给定客户创建工单时，代理可以从为该客户定义的服务列表中选择服务。
- 工单截止时间根据与客户签订的 SLA 计算。

iTop 中有两种管理服务目录的替代方案：

- 服务管理（服务、SLA、合同）
- 服务提供商的服务管理（服务、SLA、合同）

第一个用于在单一公司内管理服务，第二个专为服务提供商设计。两个模块提供相同的管理元素，区别在于这些元素之间的关系。在以下部分中，我们描述了这些差异。

## 合同类型

合同类型允许您为客户和供应商合同定义分类，以便对它们进行分组。例如：

- 硬件合同
- 维护合同
- 外包合同

### 合同类型属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |

## 客户合同

客户合同允许您定义客户已购买（请求）哪些服务以及这些服务对应的 SLA 是什么。

默认情况下，iTop 中没有定义覆盖窗口。

可以为给定客户记录多个合同。对于每个客户合同，您可以链接文档和相关的联系人。

### 客户合同属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 客户 | 外键指向组织 | 是 |
| 描述 | 多行字符字符串 | 否 |
| 开始日期 | 日期（年-月-日） | 否 |
| 结束日期 | 日期（年-月-日） | 否 |
| 成本 | 字母数字字符串 | 否 |
| 成本货币 | 可选值：美元、欧元 | 否 |
| 计费频率 | 字母数字字符串 | 否 |
| 成本单位 | 字母数字字符串 | 否 |
| 供应商 | 外键指向组织 | 是 |
| 状态 | 可选值：实施、已废弃、生产 | 否 |
| 合同类型 | 外键指向合同类型 | 否 |

### 标签页

| 标签页 | 描述 |
|--------|------|
| 联系人 | 此客户合同的所有联系人 |
| 文档 | 此客户合同的所有文档 |
| 服务 | 此合同购买的所有服务 |
| 供应商合同 | 支持此服务的所有供应商合同 |
| 依赖的配置项 | 用于提供此服务的所有配置项 |

## 创建客户合同

“客户合同”菜单提供已定义的客户合同列表。只需点击“新建”按钮即可创建新的客户合同。

“服务”标签页显示为此合同向客户提供的服务列表。您可以为每个服务指定适用的 SLA。

## 交付模型

交付模型定义哪些团队为一组组织（客户）提供支持和服务。它还可以用于记录关键人员及其角色：服务经理、首席技术官、帮助台经理。

多个客户可以共享相同的交付模型。尽管如此，定义多个交付模型可能很有用，特别是如果您想限制可以将工单分配给的团队数量。

给定客户的交付模型用于确定为该客户可以将工单分配给哪个团队。

### 交付模型属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 组织 | 外键指向组织 | 是 |
| 描述 | 多行字符字符串 | 否 |

### 标签页

| 标签页 | 描述 |
|--------|------|
| 联系人 | 此交付模型的所有联系人（团队和个人） |
| 客户 | 使用此交付模型的所有客户 |

## 创建交付模型

“交付模型”菜单提供已定义的交付模型列表。只需点击“新建”按钮即可创建新的交付模型。

## 供应商合同

您还可以链接供应商合同涵盖的配置项。对于每个客户合同，您可以链接文档和相关的联系人。

| 开始日期 | 日期（年-月-日） | 否 |
| 结束日期 | 日期（年-月-日） | 否 |
| 合同类型 | 外键指向合同类型 | 否 |
| 成本 | 字母数字字符串 | 否 |
| 成本货币 | 可选值：美元、欧元 | 否 |
| 计费频率 | 字母数字字符串 | 否 |
| 成本单位 | 字母数字字符串 | 否 |
| 供应商 | 外键指向组织 | 是 |
| SLA | 字母数字字符串 | 否 |
| 服务时间 | 字母数字字符串 | 否 |
| 状态 | 可选值：实施、已废弃、生产 | 否 |

### 标签页

|--------|------|

## 创建供应商合同

“供应商合同”菜单提供已定义的供应商合同列表。只需点击“新建”按钮即可创建新的供应商合同。

## SLA

服务级别协议（SLA）是一组 SLT。它定义了提供商与客户之间针对给定服务集的协议。

每个 IT 部门可以定义自己的 SLA 列表。

### SLA 属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 供应商 | 外键指向组织 | 是 |
| 描述 | 多行字符字符串 | 否 |

### 标签页

| 标签页 | 描述 |
|--------|------|
| SLT | 此 SLA 的所有服务级别目标 |
| 客户合同 | 使用此 SLA 的所有客户合同 |

## 创建 SLA

“SLA”菜单提供已定义的服务级别协议列表。只需点击“新建”按钮即可创建新的 SLA。

## SLT

服务级别目标（SLT）允许您定义必须遵守的协议指标。默认情况下，iTop 提供两种类型的指标：

- **接管时间（TTO）**：这是从创建工单到接管（分配给代理）之间的时间
- **解决时间（TTR）**：这是从创建工单到解决之间的时间

SLT 由以下要素定义：

- 名称
- 指标类型
- 请求类型
- 请求优先级

### SLT 属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 优先级 | 可选值：严重、高、中、低 | 否 |
| 请求类型 | 可选值：事件、服务请求 | 否 |
| 指标 | 可选值：TTO、TTR | 否 |
| 值 | 数值（可为负数） | 否 |
| 单位 | 可选值：小时、分钟 | 否 |

## 创建 SLT

“SLT”菜单提供已定义的服务级别目标列表。只需点击“新建”按钮即可创建新的 SLT。

## 服务

服务是服务目录的基础。它们用于记录一个或多个 IT 部门可以提供的所有服务。例如：

- 应用管理
- 桌面和终端用户管理
- 网络管理
- 系统管理

每个 IT 提供商可以定义自己的服务列表。服务用于描述用户请求或事件的特征。

使用增强门户时，服务必须分组到服务族中。

对于每个服务，您可以记录：

- 交付服务所需的配置项
- 服务所依赖的供应商合同

### 服务属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 供应商 | 外键指向组织 | 是 |
| 服务族 | 外键指向服务族 | 否 |
| 描述 | 多行字符字符串 | 否 |
| 状态 | 可选值：实施、已废弃、生产 | 否 |

### 标签页

| 标签页 | 描述 |
|--------|------|
| 服务子类别 | 此服务的所有子类别 |
| 文档 | 链接到服务的所有文档 |
| 联系人 | 此服务的所有联系人 |
| 客户合同 | 购买此服务的所有客户合同 |
| 供应商合同 | 支持此服务的所有供应商合同 |
| 依赖的配置项 | 用于提供此服务的所有配置项 |

## 创建服务

“服务”菜单提供服务列表。只需点击“新建”按钮即可创建新的服务。

### 服务族属性

|------|------|--------|
| 名称 | 字母数字字符串 | 是 |

### 标签页

| 标签页 | 描述 |
|--------|------|
| 服务 | 此类别中的所有服务 |

## 服务子类别

服务子类别用于更精确地定义服务。例如，您可以为服务“系统管理”定义以下子类别：

- 故障排除
- 订购新服务器
- 配置新虚拟机
- 维修服务器

服务子类别与用户请求类型（事件或服务请求）相关。这是为了自动完成用户请求或事件的分类。

### 服务子类别属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 服务 | 外键指向服务 | 是 |
| 供应商 | 映射到外部键的字段 | 是 |
| 状态 | 可选值：实施、已废弃、生产 | 否 |
| 请求类型 | 可选值：事件、服务请求 | 是 |
| 供应商名称 | 映射到外部键的字段 | 是 |
| 描述 | 多行字符字符串 | 否 |

## 创建服务子类别

“服务子类别”菜单提供服务子类别列表。只需点击“新建”按钮即可创建新的服务子类别。
