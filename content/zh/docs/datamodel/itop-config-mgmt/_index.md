## 概述

配置管理模块是 iTop 的强制模块。它包含 CMDB 所有基本构建块的定义：组织、联系人（人员和团队）以及常见的物理设备（服务器、网络设备……）、软件元素（数据库服务器、数据库架构、许可证、补丁……）以及所有这些对象之间的关系。

## 联系人

### 人员

人员类用于描述 CMDB 中的物理人员联系人。人员可以分组为团队。人员可以与其他配置项关联（例如，描述在应用程序出现问题时联系谁）。在 iTop 中，用户请求的呼叫者是一个人员，分配给解决该请求的代理也是一个人员。

#### 人员属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 姓氏 | 字母数字字符串 | 是 |
| 名字 | 字母数字字符串 | 是 |
| 组织 | 外键指向组织 | 是 |
| 状态 | 可选值：活跃、非活跃 | 是 |
| 位置 | 外键指向位置 | 否 |
| 职能 | 字母数字字符串 | 否 |
| 经理 | 外键指向人员 | 否 |
| 员工编号 | 字母数字字符串 | 否 |
| 电子邮件 | 电子邮件地址 | 否 |
| 通知 | 可选值：否、是 | 否 |
| 电话 | 字母数字字符串 | 否 |
| 手机 | 字母数字字符串 | 否 |

#### 标签页

| 标签页 | 描述 |
|--------|------|
| 团队 | 此人所属的所有团队 |
| 工单 | 此人是呼叫者的所有工单 |
| 配置项 | 与此联系人关联的所有配置项 |

#### 创建新人员

在快速创建区域输入类名以创建新对象，然后填写表单并点击创建按钮。

### 团队

团队代表一组人员。在 iTop 中，团队可用于：将一组人员关联到特定配置项，创建用于分配工单的“工作组”，或分组人员以接收通知。

要添加团队成员，请点击“成员”标签页，使用按钮从列表中添加/移除人员。您还可以为每个团队成员指定“角色”。

## 文档

iTop 中有多种类型的文档。所有文档都可以从“配置管理”部分的“文档”菜单进行管理。

### 文件文档

文件文档用于上传可在 CMDB 中多个对象之间共享的文档。文档（任何格式：Word、PDF、电子表格等）上传到 iTop 后，可以“附加”到许多共享相同文档的不同配置项。这对于处理说明、技术文档等需要从 CMDB 多个位置引用的情况非常有用。

文件上传大小限制取决于：
- Web 服务器上配置的最大上传大小
- MySQL 服务器上配置的最大 MySQL 数据包大小

#### 文件文档属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |

| 版本 | 字母数字字符串 | 否 |
| 文档类型 | 外键指向文档类型 | 否 |
| 描述 | 多行字符字符串 | 否 |
| 文件 | 任何二进制内容（文档） | 是 |

#### 标签页

| 合同 | 与此文档关联的所有合同 |
| 服务 | 与此文档关联的所有服务 |

## 配置项

### 应用解决方案

应用解决方案描述由（或依赖于）多个基本组件组成的复杂应用程序。它们是用于记录大型应用程序的专用“组”类型。应用解决方案传达的主要信息是其关系列表。

#### 应用解决方案属性

| 名称 | 类型 | 必填？ |
|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 组织 | 外键指向组织 | 是 |
| 状态 | 可选值：活跃、非活跃 | 否 |
| 业务关键性 | 可选值：高、低、中 | 否 |
| 投产日期 | 日期（年-月-日） | 否 |
| 描述 | 多行字符字符串 | 否 |

#### 标签页

| 标签页 | 描述 |
|--------|------|
| 联系人 | 此配置项的所有联系人 |
| 文档 | 链接到此配置项的所有文档 |
| 工单 | 此配置项的所有工单 |
| 配置项 | 组成此应用解决方案的所有配置项 |
| 业务流程 | 依赖此应用解决方案的所有业务流程 |
| 供应商合同 | 此配置项的所有供应商合同 |
| 服务 | 受此配置项影响的所有服务 |

#### 创建新应用解决方案

在快速创建区域输入类名以创建新对象，然后填写表单并点击创建按钮。

要将其他配置项链接到新的应用解决方案，请点击“配置项”标签页，使用按钮从列表中添加/移除配置项。“联系人”标签页用于管理与联系人的关系，以此类推。

#### 关系

- **影响**：应用解决方案被认为影响其所有相关的业务流程（列在“业务流程”标签页中）。
- **依赖于**：应用解决方案被认为依赖于其所有相关的配置项（列在“配置项”标签页中）。

### 业务流程

业务流程用于记录对运营至关重要的高级流程或重要应用程序。它与应用解决方案非常相似，但用于描述组织中的更高级别应用程序或整个流程。业务流程传达的主要信息是其与应用解决方案和联系人的关系列表。

#### 业务流程属性

|------|------|--------|
| 名称 | 字母数字字符串 | 是 |
| 组织 | 外键指向组织 | 是 |
| 状态 | 可选值：活跃、非活跃 | 否 |
| 业务关键性 | 可选值：高、低、中 | 否 |
| 投产日期 | 日期（年-月-日） | 否 |
| 描述 | 多行字符字符串 | 否 |

#### 标签页

| 标签页 | 描述 |
|--------|------|
| 联系人 | 此配置项的所有联系人 |
| 文档 | 链接到此配置项的所有文档 |
| 工单 | 此配置项的所有工单 |
| 应用解决方案 | 影响此业务流程的所有应用解决方案 |

#### 创建新业务流程

业务流程对象的创建与应用解决方案的创建类似，不同之处在于其组件是应用解决方案而不是任何配置项。

#### 关系

- **影响**：业务流程不会额外影响其他配置项。
- **依赖于**：业务流程被认为依赖于其所有相关的应用解决方案（列在“应用解决方案”标签页中）。

| 操作系统版本 | 计算机“操作系统版本”的可能值。iTop 中的操作系统版本按操作系统系列组织。例如，如果定义了操作系统类型“MacOS X”，则该类型的版本可以是：10.0 - Cheetah、10.1 - Puma、10.2 - Jaguar、10.3 - Panther、10.4 - Tiger 等。而对于操作系统类型“Windows”，版本可以是：XP、XP SP1、XP SP2、XP SP3、Vista、Vista SP1、Vista SP2、7、7 SP 1、Server 2008、Server 2008 R2、Server 2012 等。 |
