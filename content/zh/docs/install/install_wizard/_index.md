## 安装向导

- 确保您有一个配置正确的 Apache/PHP 实例正在运行。
- 将 "web" 目录的内容解压到 Web 服务器提供服务的目录中（例如 Linux 上的 `/var/www/html/itop` 或 Windows/IIS 上的 `C:\inetpub\wwwroot\itop`）。
- 设置适当的权限（请参阅[专用 Wiki 页面](https://www.itophub.io/wiki/page?id=3_2_0:install:itop_setup_protection)）。
- 将 Web 浏览器指向文件已解压目录对应的 URL（例如 `http://localhost/itop`），然后按照屏幕上的指示操作。

iTop 提供了一个分步向导来安装应用程序。

### 第一步：检查配置和先决条件

安装的第一步是检查配置的一致性、Web 服务器用户的权限以及 MySQL、PHP 和可选 PHP 扩展的先决条件。

### 第二步：选择安装类型

第二步提供选择：升级现有 iTop 实例或从头开始安装应用程序。

如果您选择安装新实例，向导的第三步将提示您接受许可协议。

### 第四步：数据库连接信息

向导的第四步提示您输入连接 MySQL 的凭据以及用于安装 iTop 的数据库。请确保提供一个具有足够权限的账户来安装和运行 iTop。此账户需要在数据库中创建表、触发器和视图，以及在升级时删除视图。如果您选择让安装程序为您创建新数据库，此账户需要有足够的权限来执行此操作。

您还可以指定一个前缀，该前缀将添加到 iTop 在数据库中创建的所有表和视图的名称中。如果您想在同一个 MySQL 数据库中安装多个 iTop，这很有用，但建议避免这样做。最好创建多个数据库。

请注意，数据库服务器不需要与 PHP/Web 服务器在同一台物理服务器上；如果您喜欢采用双层架构，或者连接到已安装的 MySQL 服务器，它可以是远程主机。

### 管理员账户

向导的下一步提示您输入管理员账户的信息。iTop 认证可以依赖外部机制（LDAP、Active Directory 或 JASIG-CAS），但安装要求至少创建一个具有本地密码的管理员账户。

### 其他信息

下一个屏幕提示您输入几个其他信息：

- **默认语言**：这是登录屏幕的语言以及新创建用户账户的默认语言。
- **访问 iTop 的 URL**：如果用户将通过不同的 URL 访问 iTop（通过 DNS 别名、不同的 IP 地址等），请更改此值。
- **示例数据**：如果您想快速体验 iTop，请安装示例数据。示例数据提供一些示例配置项（服务器、应用程序、联系人）以及服务目录的默认设置，以确保应用程序的快速部署。如果您正在部署 iTop 的生产实例并想要一个"干净"的安装，请不要选择此选项。

### 配置 CMDB

CMDB 由 5 个模块组成：

- **配置管理核心模块**：始终安装。此模块提供 iTop 中必须始终存在的基本对象：组织、位置、联系人以及一些常见的 CI，如服务器、组、应用程序等。
- **数据中心管理模块**：提供用于管理数据中心物流的配置项：机架、机箱、电源（PDU、电源...）。如果您不管理数据中心，可能不需要此模块。
- **终端用户设备模块**：定义与终端用户相关的设备：电话、PC、平板电脑... 如果您不计划在 iTop 中管理此类设备，可以取消选中此选项。
- **高级存储管理模块**：定义与存储设备管理相关的配置项：SAN 交换机、NAS、磁带库和磁带等。它是可选的。
- **虚拟化管理模块**：定义虚拟机、虚拟机管理程序、集群等。如果您不计划在 iTop 中管理虚拟设备，可以取消选中它。

### 配置服务管理

iTop 中的服务管理有两种风格。选择最能代表您情况的选项：

- **企业服务管理**：最适合描述同一基础设施用于向一组客户交付服务的情况。客户可以是同一组织的内部实体或外部客户。
- **服务提供商服务管理**：最适合用于交付/支持客户的基础设施对每个客户是特定的情况，客户之间几乎没有重叠（共享）。

这两个模块的文档集中在：[服务管理（服务、SLA、合同）模块](https://www.itophub.io/wiki/page?id=3_2_0:datamodel:itop-service-mgmt)。

### 配置工单管理

如果您计划在 iTop 中管理工单，基本上有两个选项：

- **简单工单**：最适合实现简单流程，其中事件和用户请求之间几乎没有区别。所有工单共享相同的生命周期，您可以随时更改工单的分类。如果您计划让最终用户通过门户直接创建工单，无论是提交服务请求还是报告事件，这个选择可能是合适的。
- **ITIL 合规工单**：最适合实现严格的 ITIL 合规流程。用户请求和事件工单之间有明确的区别。（您可以选择仅部署其中一个流程或两者都部署。）

当然，您也可以选择不在 iTop 中管理此类工单。在这种情况下，选择第三个选项。

### 配置变更管理

如果您计划在 iTop 中实施变更管理，也有两个选项：

- **简单变更管理**：提供一种单一类型的通用变更请求工单。
- **ITIL 合规变更管理**：提供完整的 ITIL 合规变更流程集，包括标准、例行和紧急变更工单。

您也可以通过选择第三个选项来完全停用 iTop 中的变更管理。

### 其他 ITIL 流程

除了变更和事件管理之外，还可以通过选择相应的工单类型来部署另外两种 ITIL 合规流程：

- **问题管理**
- **已知错误和 FAQ**

### 启动安装

完成所有选择后，将显示摘要。您可以点击每个部分的"加号"按钮来查看您的选择。

要开始实际的 iTop 安装，请点击"安装！"按钮。

安装进度将显示一个移动的进度条。

安装完成后，您可以：通过点击"工单管理手动说明"链接查看关于设置后台任务的说明。或者通过点击"进入 iTop"按钮立即进入 iTop。

恭喜，您已成功安装 iTop！

### 自动化安装

iTop 的安装通常使用 "setup" 页面以交互方式进行。然而，一旦安装页面收集了有关所需配置的信息，实际的安装过程就是完全无人值守的，因此可以自动化。

### 安装时的主题预编译

此功能是安装的一个优化，可以显著节省主题处理时间。

有关更多详细信息，请参阅[主题预编译](../../customization/how-to/user-interface/theme/)。
